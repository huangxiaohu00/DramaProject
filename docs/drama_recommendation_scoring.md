# 剧集推荐系数算法

本文记录后续推荐题材加权时的剧集推荐系数计算规则。

## 标签推荐权重

标签推荐权重来自 Mobo 订单偏好分析结果，范围为 `0~1`。

当前权重口径：

```text
金额权重 = 0.85
订单数权重 = 0.15
```

每个分类内排名第 1 的题材标签权重为 `1.0`，其他标签按金额、订单数、样本量平滑后归一化到 `0~1`。

当前用于偏好分析的泛化标签排除集：

```text
现代言情
古代言情
现代
现代都市
```

## 剧集多标签合成

一个剧集可能有 1 个或多个有效标签。计算剧集推荐系数时，不做简单相加，也不做简单平均。

推荐规则：

```text
1. 取该剧集命中的有效题材标签
2. 查询这些标签在当前维度下的推荐权重
3. 按推荐权重从高到低排序
4. 只取前 3 个标签
5. 使用 1、0.5、0.25 递减系数合成
6. 最终封顶 1.2
```

公式：

```text
drama_score = min(
    tag_weight_1 * 1.0
  + tag_weight_2 * 0.5
  + tag_weight_3 * 0.25,
  1.2
)
```

处理原则：

- 单标签剧集最高通常为 `1.0`。
- 多标签剧集如果多个标签都强，可以略高于单标签剧集。
- 多标签剧集最高封顶 `1.2`，避免因为标签数量多而天然占优。
- 第 4 个及以后的标签不参与计算。
- 没有权重的标签跳过。

## 标签权重查找优先级

给定剧集的语言、剧场、标签，按以下顺序查找标签推荐权重：

```text
1. 语言 + 剧场标签权重
2. 语言标签权重
3. 剧场标签权重
4. 全局标签权重
5. 默认兜底
```

如果高优先级没有找到该标签权重，就继续向下查找。

## 无标签或无权重兜底

如果剧集没有匹配到标签，或者标签存在但所有标签都找不到可用权重，则使用默认兜底推荐系数。

默认值：

```text
drama_score = 0.5
```

状态建议：

```text
无标签兜底
标签无权重兜底
标签命中
```

处理原则：

- 不给 `0`，因为无标签不等于不适合推荐。
- 不给 `1`，因为无标签不应等同于强偏好题材。
- `0.5` 作为中性偏弱的冷启动分。

## 示例

单标签剧集：

```text
标签：总裁
总裁权重 = 1.0

drama_score = 1.0
```

双标签剧集：

```text
标签：总裁, 甜宠
总裁权重 = 1.0
甜宠权重 = 0.45

drama_score = min(1.0 + 0.5 * 0.45, 1.2)
            = 1.2
```

多标签弱偏好剧集：

```text
标签：甜宠, 逆袭, 家庭亲情
甜宠权重 = 0.45
逆袭权重 = 0.35
家庭亲情权重 = 0.30

drama_score = 0.45 + 0.5 * 0.35 + 0.25 * 0.30
            = 0.70
```

无标签剧集：

```text
drama_score = 0.5
tag_status = 无标签兜底
```
