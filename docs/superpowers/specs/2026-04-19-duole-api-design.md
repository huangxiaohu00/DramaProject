# 多乐剧集接口独立脚本设计

**日期**: 2026-04-19

## 目标

新增一个独立 Python 脚本，请求多乐剧集接口，并把响应结果结构化为固定字段，便于人工核对和后续复用。

## 范围

- 新增独立脚本 `duole_api.py`
- 保留并输出以下字段：
  - `剧集ID`
  - `剧集名称`
  - `剧场名称`
  - `剧集标签`
  - `语言`
  - `推送时间`
- 默认使用当前已验证可用的 URL、请求头、请求体
- 运行脚本时直接发起真实请求并打印结构化结果

## 数据映射

- `剧集ID`: 优先 `story_id`，其次 `id`，再其次 `_id`
- `剧集名称`: 优先 `story_name`，其次 `name`
- `剧场名称`: `type`
- `剧集标签`: `c_name`
- `语言`: 优先使用响应中的 `language` 做语言映射；缺失时回退到 `lang_id`
- `推送时间`: `publish_time`

## 结构

- `build_duole_headers()`：返回请求头
- `build_duole_payload()`：返回请求体
- `fetch_duole_response()`：执行 HTTP 请求并返回 JSON
- `normalize_duole_items()`：把原始列表转换为固定结构
- `main()`：串起请求、转换、打印

## 错误处理

- HTTP 非 200 时抛出异常
- 响应 `success` 为假时抛出异常
- 响应结构异常时按空列表处理
