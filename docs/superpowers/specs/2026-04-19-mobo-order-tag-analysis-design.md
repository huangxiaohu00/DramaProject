# Mobo 订单标签偏好分析设计

**日期**: 2026-04-19

## 目标

基于订单文件和 Mobo 平台剧集接口，补齐订单对应剧集的 Mobo 标签，并输出三类标签偏好排序：

1. 不同语言用户偏好什么标签
2. 不同剧场用户偏好什么标签
3. 不同语言 + 不同剧场用户偏好什么标签

排序规则统一为：

1. 按 `金额` 降序
2. 金额相同时按 `订单数` 降序

## 范围

- 只使用 Mobo 平台数据
- 不使用多乐接口
- 不改写原始订单文件
- 生成一个新的分析结果 Excel

## 输入

### 订单文件

文件：`出单数据.xlsx`

当前字段：

- `语言`
- `剧场`
- `剧名`
- `订单数`
- `金额`

### Mobo 数据源

复用以下文件中的既有逻辑和配置：

- `merged_sucai_gui.py`
- `build_platform_catalogs.py`

核心接口：

- `https://kocserver-cn.cdreader.com/api/v1/res/getlistpc`

核心可用字段：

- `agencyName`：剧场
- `seriesName`：剧名
- `languageName`：语言
- `seriesTypeList`：剧集标签
- `seriesId`：剧集 ID

## Mobo 全量抓取策略

按语言拉取 Mobo 数据。

### 语言范围

至少覆盖订单表中出现的语言，并补齐订单口径到 Mobo 口径的映射：

- `翻译英语` -> `英语`

其余语言按原值直接匹配。

### 剧场范围

抓取 Mobo 接口返回的所有剧场，不预先按剧场过滤。

### 分页策略

对每个语言分页请求：

- 从 `pageIndex = 1` 开始
- `pageSize = 10000`
- 当页返回为空或返回条数小于 `pageSize` 时停止

### 接口口径

只取一个稳定的 `getlistpc` 请求口径，不再分别抓取多个 `orderType`。

原因：

- 已确认不同 `orderType` 全量拉取后的总量一致
- 本次目标是获取全量剧集及标签，不做推荐位分析

默认使用与 `merged_sucai_gui.py` 一致的接口口径：

- `platform = None`
- `audioType = 0`
- `localType = 0`
- `projectType = 2`
- `orderType = 0`

## 标准化与匹配

### 剧名标准化

复用 `merged_sucai_gui.py` 中的 `normalize_title()`。

### 语言标准化

订单语言先做标准化，再与 Mobo 的 `languageName` 匹配。

当前明确规则：

- `翻译英语` -> `英语`

### 剧场标准化

订单剧场与 Mobo 剧场统一到展示名口径后再匹配。

明确映射：

- `moboreels` -> `MoboReels`
- `shortmax` -> `ShortMax`
- `flareflow` -> `FlareFlow`
- `honeyreels` -> `HoneyReels`
- `snackshort` -> `SnackShort`
- `dramabox` -> `DramaBox`
- `flickreels` -> `FlickReels`
- `kalostv` -> `KalosTV`
- `starshort` -> `StarShort`

订单中已是展示名的值保持不变。

### 匹配键

订单和 Mobo 数据使用以下组合键精确匹配：

- `标准化语言`
- `标准化剧场`
- `标准化剧名`

不做模糊匹配。

## 标签获取与归一化

Mobo 原始标签直接取 `seriesTypeList`。

但 `seriesTypeList` 可能存在：

- 中文
- 繁体中文
- 德语、意大利语、葡萄牙语等外语
- 个别编码异常或大小写、空格差异

最终分析时，标签统一转换为中文标准标签。

规则：

- 如果 `seriesTypeList` 是数组，先去空、去重
- 对每个原始标签执行标准化：
  - 去前后空格
  - Unicode 归一化
  - 修正常见编码差异
  - 繁体标签转换到简体中文口径
  - 外语标签通过别名字典映射到中文标准标签
- 同一剧在多页命中时，标准中文标签做并集
- 输出到订单表时，将中文标准标签拼接为逗号分隔字符串，列名为 `Mobo标签`

### 标签中文归一化策略

分析时只使用中文标准标签。

实现上增加一份“标签别名字典”，把外语或异常形式映射到中文，例如：

- `CEO` -> `总裁`
- `Verborgene Identität` -> `隐藏身份`
- `L'amore di contratto` -> `契约婚姻`
- `Dolce Romanzo` -> `甜宠`

如果某个原始标签未命中别名字典：

- 先尝试通过繁简转换和编码清洗归一化
- 仍无法归一化为中文时，写入单独的检查表供人工补充

未匹配到 Mobo 剧集时：

- `Mobo标签` 留空
- 同时写入 `未匹配订单` sheet 便于人工核对

## 标签拆分分析口径

### 标签拆分

将 `Mobo标签` 按单个标签拆开，例如：

- `["女频", "闪婚", "失忆", "现代言情", "总裁"]`

### 统计口径

每条已匹配订单拆分后，对其包含的每个标签都记一次完整贡献，不做均分。

例如一条订单：

- 订单数 = 3
- 金额 = 147.28
- 标签 = `总裁,失忆,现代言情`

则这 3 个标签各自累计：

- `订单数 +3`
- `金额 +147.28`

这是本次分析的固定口径。

## 输出

生成新文件：

- `platform_catalog_output/mobo_order_tag_analysis.xlsx`

包含以下 sheet：

### 1. `订单打标`

基于原始订单字段新增：

- `标准化语言`
- `标准化剧场`
- `标准化剧名`
- `Mobo标签`
- `Mobo剧集ID`
- `Mobo剧名`
- `Mobo剧场`

### 2. `按语言偏好标签`

字段：

- `语言`
- `标签`
- `金额`
- `订单数`
- `排名`

排序：

- `语言` 分组内按 `金额 desc`、`订单数 desc`

### 3. `按剧场偏好标签`

字段：

- `剧场`
- `标签`
- `金额`
- `订单数`
- `排名`

排序：

- `剧场` 分组内按 `金额 desc`、`订单数 desc`

### 4. `按语言剧场偏好标签`

字段：

- `语言`
- `剧场`
- `标签`
- `金额`
- `订单数`
- `排名`

排序：

- `语言 + 剧场` 分组内按 `金额 desc`、`订单数 desc`

### 5. `未匹配订单`

保留未命中 Mobo 标签的订单明细，便于检查缺失原因。

### 6. `标签归一化检查`

用于核对外语、繁体或异常编码标签最终被归一化成了什么中文标签。

## 实现边界

- 新增独立分析脚本，不改现有 GUI 主流程
- 优先复用现有 Mobo 配置、语言映射、剧名标准化逻辑
- 不引入新的外部依赖

## 校验要求

完成后至少验证：

1. 能成功抓取 Mobo 全量分页数据
2. 能生成带 `Mobo标签` 的订单明细
3. 三张分析表都存在并按金额、订单数倒序
4. 存在未匹配订单时能正确落到 `未匹配订单` sheet
5. 标签最终以中文口径输出，未归一化标签会被记录到 `标签归一化检查` sheet
