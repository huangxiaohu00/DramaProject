# 推荐评分调整实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将今日剧集推荐从旧的“来源信号分 * 标签分”排序，升级为“分层出榜 + 层内综合打分 + 题材/内容系数”的绝对收入导向算法。

**Architecture:** `8_recommendation_rules.py` 负责候选去重、爆标签计算、A~G 分层、层内主分计算、内容形态系数和最终取值；`8_tag_scoring.py` 继续读取 `basic_data/drama_theme_tag_analysis.xlsx`，但输出改为 `0.8~1.2` 的题材系数，默认值为 `1.0`；`2_config.py` 保存剧龄、爆发、题材、内容系数等阈值；`6_today_recommend.py` 负责编排候选采集、每一步数据量统计、按语言剧场上限取值，并在默认配置下将素材视为全部达标且不请求素材接口。

**Tech Stack:** Python dataclass、openpyxl、unittest。

---

## 评分口径

### 目标与来源职责

- 目标函数：`绝对收入最大化`。
- `Mobo 新剧`、`北斗新剧`：判断剧龄与新剧标签。
- `Mobo 推荐`：判断当前推荐强度；`Top 1~50 / 51~100 / 101~500` 分别对应新爆辅助、中爆辅助、老爆辅助。
- `多乐按天推荐剧单`、`多乐 DramaBox 英语榜`、`多乐 DramaBox 小语种榜`：都先抓取 `1000` 行原始数据，再分别裁切为 `600 / 200 / 400`，作为日期型爆发证据。
- `飞书热门链接日榜`：按天分段解析；首列为日期的行才算有效推荐行，首列为 `推荐日期` 的行为表头，其余备注行忽略。
- `飞书北斗英语热榜`：保留英语热榜，作为北斗英语爆发证据。
- `北斗收入榜`：仅取非英语，作为收入验证来源，同时补充 `<=200 且 <=28天`、`<=400 且 <=60天`、`<1000` 三档爆标签辅助；与飞书北斗英语热榜合并后构成完整的“北斗爆款池”。
- `飞书已发布`：硬过滤。

### 剧龄与爆标签

剧龄：

- `1~14天 = 新`
- `15~28天 = 中`
- `29天+ = 老`

爆标签：

- `新爆`
  - 近 `1~14天` 命中日期型热榜/推荐榜；或
  - `Mobo 推荐 Top 50`；或
  - 北斗 `rank <= 200` 且 `上新 <= 28天`。
- `中爆`
  - 未命中新爆，且近 `15~28天` 命中日期型热榜/推荐榜；或
  - `Mobo 推荐 51~100`；或
  - 北斗 `rank <= 400` 且 `上新 <= 60天`。
- `老爆`
  - 未命中新爆/中爆，且满足任一：
  - `Mobo 推荐 101~500`
  - 北斗 `rank < 1000`

### 最终分层

- `A：新 + 新爆`
- `B：中 + 新爆`
- `C：新 + 其他`
- `D：中 + 中爆`
- `E：中 + 其他`
- `F：老剧爆款剩余池`（剧龄 `29天+` 且命中任一爆标签）
- `G：普通剩余剧`（未命中任何爆标签）

出榜顺序固定为：`A > B > C > D > E > F > G`。

### A~E 层内主分

```text
层内主分 = 45% 爆发推荐分 + 35% 新鲜度分 + 20% 收入验证分
最终层内分 = 层内主分 × 题材系数 × 内容形态系数
```

说明：

- `爆发推荐分 = 0.6 × 日期型爆发分 + 0.25 × Mobo推荐分 + 0.15 × 北斗爆标签分`
- `日期型爆发分`
  - 近14天命中 `>=2` 个独立榜单来源：`100`
  - 近14天命中 `1` 个独立榜单来源：`85`
  - 15~28天命中 `>=2` 个独立榜单来源：`70`
  - 15~28天命中 `1` 个独立榜单来源：`55`
  - 未命中：`0`
- `Mobo推荐分`
  - `Top 1~50 = 100`
  - `51~100 = 70`
  - `101~500 = 40`
  - 其他 = `0`
- `北斗爆标签分 / 收入验证分`
  - 北斗 `rank <= 200 且 <=28天 = 100`
  - 北斗 `rank <= 400 且 <=60天 = 70`
  - 北斗 `rank < 1000 = 40`
  - 未命中 = `0`
- `新鲜度分`
  - `1~14天 = 100`
  - `15~28天 = 70`
  - `29天+ = 30`

### F、G 层特殊规则

F 层：

```text
F层内分 = 55% 爆发推荐分 + 15% 新鲜度分 + 30% 收入验证分
最终F分 = F层内分 × 题材系数 × 内容形态系数
```

- F 层内部优先级：`老剧 + 新爆标签 > 老剧 + 中爆标签 > 老剧 + 老爆标签`。

G 层：

```text
G层排序 = 新推广时间优先 > 题材系数 > 内容形态系数 > 弱推荐信号
```

### 题材系数

- 无题材：`1.0`
- 有题材：`0.8 ~ 1.2`
- 题材偏好按语言维度的历史收入金额建模，订单数只用于样本置信度，不再直接参与偏好方向。
- 题材系数只允许影响同层内排序，不允许跨层翻盘。

### 出榜配额

- 目标语言固定为：`英语 / 法语 / 葡萄牙语 / 德语 / 繁体中文 / 俄语 / 意大利语`。
- 每个语言继续按 `LANGUAGE_THEATER_QUOTAS` 控制，总目标按配额自动计算，当前为 `66` 部。
- 每个语言内继续使用 `LANGUAGE_THEATER_QUOTAS` 控制各剧场上限，不额外增加新的 `Moboreels <= 4` 全局限制。

### 内容形态系数

- 真人：`1.0`
- AI：`0.8`
- 漫剧：`0.6`
- AI + 漫剧：`0.6`
- 成人裸露：直接过滤，不参与排序。

### 调试统计口径

- 每个来源在进入规则前都要记录：`原始返回条数`、`解析成功条数`、`按语言拆分后的条数`。
- 关键处理中间态继续记录：`去重后条数`、`飞书已发布过滤后条数`、`成人拦截条数`、`A~G 各层条数`、`素材达标条数 / 跳过条数`、`最终入榜条数`。
- 统计结果需要按 `来源 / sheet / 语言 / 剧场 / 阶段` 可追溯，便于快速定位是“没取到数据”还是“后续过滤过严”。
- 输出形式至少包含一份独立调试统计文件，避免只看最终推荐表才发现数量不足。
